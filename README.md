# note CodeBlock Visualizer 🚀

note.comのコードブロックをモダンで読みやすいスタイルにカスタマイズし、ユーティリティ機能を追加するブラウザ拡張機能（Chrome/Edge/Firefox対応、Manifest V3）です。

---

## ✨ 主な機能

| 機能 | 詳細 |
| :--- | :--- |
| **🎨 5つの配色テーマ** | One Dark Pro, Dracula, Monokai, GitHub Dark, GitHub Light から選択可能。 |
| **🔤 プログラミングフォント** | JetBrains Mono, Fira Code, Source Code Pro, System Monospace に対応。 |
| **📏 サイズと高さ調整** | 11px〜22pxの範囲でフォントサイズをスライダー調整可能。 |
| **📋 ワンクリックコピー** | コードブロック右上にアニメーション付きの「Copy」ボタンを設置。 |
| **🔢 行番号の表示切替** | 行番号の表示・非表示をトグルスイッチで切り替え可能。 |
| **⚡ リアルタイム反映** | 設定ポップアップで変更した内容は、開いているタブに即座に反映されます。 |

---

## 🛠️ 技術的な特徴

### 1. SPA（React）との競合回避
note.comは単一ページアプリケーション（SPA）であるため、DOMの要素を単純に削除または書き換えるとReactの再レンダリング時にエラー（Reconciliation Crash）が発生するリスクがあります。
本拡張機能では、元の `<pre>` 要素を `display: none !important;` で非表示化しつつ、その直後にクローンしたカスタムラッパーを挿入します。また、元のコードがSPA遷移や更新等によって変化した場合も、`MutationObserver` がそれを検知してクローンにリアルタイム同期します。

### 2. 強力なスタイル孤立化 (Insulated CSS)
note.comのグローバルなスタイル（`pre[data-name="preCode"]` や `.hljs`）が `!important` で強制適用されているため、クローン側では `<pre>` や `<code>` を使わず、あえて `<div>` タグ（`.note-cloned-pre`, `.note-code-block`）に変換しています。これにより、既存のCSSによる予期せぬスタイルの崩れやオーバーライドを完全にシャットアウトし、選んだテーマ通りの見た目を保証します。

## 📦 ファイル構成

本プロジェクトは以下のファイルで構成されています。
* [manifest.json](manifest.json): 拡張機能のメタデータと権限設定 (Manifest V3)
* [content.js](content.js): DOMを監視し、コードブロックを安全にクローン＆拡張するコンテンツスクリプト
* [content.css](content.css): 各テーマの配色定義、レイアウト、Google Fontsのインポート
* [popup.html](popup.html) / [popup.css](popup.css) / [popup.js](popup.js): 設定用ポップアップ画面 (グラスモーフィズムデザイン)
* [test_page.html](test_page.html): 拡張機能を入れる前にブラウザ上で動作を確認できるローカルテスト用サンドボックス
* `icons/`: 拡張機能アイコン (16px, 48px, 128px)

## 🚀 インストール・使用方法 (初心者向けガイド)

拡張機能をブラウザに導入する方法をステップ・バイ・ステップで解説します。

---

### 📂 ステップ 1: コード（ファイル群）のダウンロード

Gitを使わない方でも、以下の方法で簡単に導入できます。

1. GitHubのページ右上にある緑色の **「Code」** ボタンをクリックします。
2. メニューから **「Download ZIP」** をクリックしてファイルをダウンロードします。
3. ダウンロードした ZIP ファイル（`note_codeblock-main.zip` など）を、パソコン上の分かりやすい場所（デスクトップなど）に解凍（展開）します。

---

### 💻 ステップ 2: 各ブラウザへのインストール手順

お使いのブラウザに合わせて以下の手順を行ってください。

#### 🟢 Google Chrome の場合
1. Chromeを開き、アドレスバーに `chrome://extensions/` と入力してエンターキーを押します（または、右上のメニュー「⋮」＞「拡張機能」＞「拡張機能を管理」を開きます）。
2. 画面右上にある **「デベロッパー モード」** のスイッチを **オン** にします。
3. 画面左上に出現する **「パッケージ化されていない拡張機能を読み込む」** ボタンをクリックします。
4. ステップ 1 で解凍したフォルダ（`manifest.json` が入っているフォルダ）を選択します。
5. インストールは完了です！

#### 🔵 Microsoft Edge の場合
1. Edgeを開き、アドレスバーに `edge://extensions/` と入力してエンターキーを押します。
2. 画面左側メニューの下部にある **「開発者モード」** のスイッチを **オン** にします。
3. 画面右上の **「展開して開発された拡張機能を読み込む」** ボタンをクリックします。
4. ステップ 1 で解凍したフォルダを選択します。
5. インストールは完了です！

#### 🦊 Firefox の場合
1. Firefoxを開き、アドレスバーに `about:debugging` と入力してエンターキーを押します。
2. 左側のメニューで **「この Firefox (This Firefox)」** をクリックします。
3. 一時的な拡張機能の項目にある **「一時的なアドオンを読み込む... (Load Temporary Add-on...)」** ボタンをクリックします。
4. 解凍したフォルダの中にある **`manifest.json`** ファイルを選択します。
5. インストールは完了です！
   > ⚠️ **注意 (Firefoxのみ)**: 
   > Firefoxの「一時的なアドオン」機能の仕様上、**Firefoxを再起動すると拡張機能が自動的に削除されます。** 再起動した場合は、上記の手順で再度 `manifest.json` を読み込ませる必要があります。

---

### 🎨 ステップ 3: 設定の変更と使い方

1. ブラウザの右上にあるパズルピースのマーク（拡張機能ボタン）をクリックし、**「note CodeBlock Visualizer」** を選択します。
2. 設定ポップアップが開くので、お好みのテーマ、フォント、サイズを設定してください。
3. [note.com](https://note.com) の任意の記事ページを開く（または開いていたページを**リロード**する）と、カスタマイズされた美しいコードブロックが表示されます。
4. 設定ポップアップからテーマなどを切り替えると、開いているページのコードブロックに**リロードなしで即座に反映**されます。

---

### 🧪 動作確認用サンドボックス (テスト用ページ)
拡張機能をブラウザに読み込ませる前に、デザインや挙動を手軽にプレビューしたい場合は、フォルダ内の [test_page.html](test_page.html) ファイルをダブルクリックしてブラウザで開いてみてください。
ページ下部にあるシミュレーターを使って、テーマ切り替えやフォントサイズ変更のテストができます。

---

### ❓ 動かないときのトラブルシューティング
* **デザインが適用されない / 白黒のままで変わらない**
  👉 拡張機能をインストールするより前に開いていた [note.com](https://note.com) のページには自動で反映されません。ページを一度 **リロード (F5キー)** してください。
* **設定を切り替えてもリアルタイムに変わらない**
  👉 拡張機能のファイルを書き換えたり修正した直後は、ブラウザにキャッシュが残っている場合があります。拡張機能の管理画面から「再読み込み (リロード)」ボタンを押し、対象のタブもリロードしてください。
